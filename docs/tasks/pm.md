## 🔄 核心职责

### 1. 项目日报 (每天 10:00 自动发送)

**触发时间:** 每天早上 10:00 (`0 10 * * *`)  
**推送方式:** 飞书消息  
**统计范围:** 过去 24 小时

### 2. 项目监控 (每次 Cron 触发必做)

**检查清单:**
- [ ] 读取所有角色任务文件 (`designer.md`, `reviewer.md`, `coder.md`, `tester.md`)
- [ ] 检查是否有任务长期处于"进行中"状态 (>30 分钟无进展)
- [ ] 检查是否有任务依赖已完成的先决条件但未开始
- [ ] 检查 Git 提交记录，确认各角色是否有新提交
- [ ] 检查 Cron 执行状态，确认是否有连续错误

**阻塞判定标准:**
- 任务"进行中"状态持续 >30 分钟无更新
- 任务"待开始"状态，但依赖已完成 >10 分钟
- Cron 连续错误 >3 次
- Git 提交停滞 >1 小时

### 3. 问题处理流程

发现阻塞 → 尝试自动解决 → 成功则更新状态，失败则飞书通知用户

### 4. 飞书通知机制

**通知条件:**
- 自动解决失败
- 任务阻塞 >30 分钟
- Cron 连续错误 >5 次
- 需要用户决策的问题
