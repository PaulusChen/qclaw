# qclaw-pm 定期项目监控配置

**创建日期:** 2026-03-06  
**优先级:** P0 (最高)

---

## 📋 监控职责

**qclaw-pm 核心职责:**
1. **定期项目状态检查** - 每 30 分钟检查所有 Agent 进度
2. **发现阻塞立即协调** - 识别问题并解决
3. **任务分发与解锁** - 完成后解锁下游任务
4. **进度汇报** - 定期向用户汇报项目进展

---

## ⏰ 监控频率

### 高频监控 (每 5 分钟)
- 检查任务文件状态更新
- 查看 git 提交记录
- 发现任务完成立即解锁下游

### 中频监控 (每 30 分钟)
- 详细检查每个 Agent 进度
- 对比预期时间线
- 发现延迟立即提醒

### 低频监控 (每天 10:00)
- 生成项目日报
- 汇总当日进展
- 规划次日任务

## 📊 监控检查清单

### 每次监控必查项

**1. Coder 状态**
- [ ] 当前任务是否进行中
- [ ] 是否超出预期时间
- [ ] 是否有阻塞问题
- [ ] git 提交是否正常

**2. Tester 状态**
- [ ] 当前测试是否执行中
- [ ] 测试报告是否提交
- [ ] 是否有 Bug 需要协调
- [ ] git 提交是否正常

**3. Designer 状态**
- [ ] 设计任务是否进行中
- [ ] 是否提交审查请求
- [ ] 审查意见是否处理
- [ ] git 提交是否正常

**4. Reviewer 状态**
- [ ] 审查任务是否进行中
- [ ] 审查报告是否提交
- [ ] 是否有待审查任务
- [ ] git 提交是否正常

---

## 🚨 阻塞识别与处理

### 阻塞判定标准

**Coder 阻塞:**
- 任务"进行中"状态持续 > 2 小时无更新
- 任务"待开始"状态，但依赖已完成 > 30 分钟
- git 提交停滞 > 1 小时

**Tester 阻塞:**
- 测试任务停滞 > 1 小时无进展
- 等待 Coder 修复 Bug > 30 分钟
- 测试环境异常 > 15 分钟

**Designer 阻塞:**
- 设计任务停滞 > 2 小时
- 等待 Reviewer 审查 > 1 小时
- 审查意见处理 > 1 小时

**Reviewer 阻塞:**
- 审查任务停滞 > 1 小时
- 等待 Designer 提交 > 30 分钟

---

### 阻塞处理流程

```
发现阻塞 → 尝试自动解决 → 成功则更新状态，失败则飞书通知用户
```

**自动解决场景:**
- 任务状态未更新 → 提醒 Agent 更新
- 依赖已解锁但未开始 → 通知开始执行
- 文档路径错误 → 协助修正

**需要用户决策场景:**
- 技术路线选择
- 优先级调整
- 资源不足
- 严重 Bug 需要修复
