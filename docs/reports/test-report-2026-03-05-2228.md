## 🔧 TEST-UNIT-FIX 修复记录

### 问题描述
`test_moving_average.py::TestEdgeCases::test_insufficient_data` 测试失败

**错误信息:**
```
assert pd.isna(result["ma5"]).all()
E   assert np.False_
E   +  where 0     True\n1     True\n2     True\n3     True\n4    False\nName: ma5, dtype: bool.all
```

### 根本原因
测试假设 MA5 在 5 行数据时全部为 NaN，但实际行为是：
- 前 4 行 (索引 0-3): NaN (数据不足)
- 第 5 行 (索引 4): 有计算值 (数据刚好足够)

### 修复方案
更新测试断言以匹配正确的业务逻辑:
```python
# 修复前
assert pd.isna(result["ma5"]).all()

# 修复后
assert pd.isna(result["ma5"]).iloc[:4].all()  # 前 4 个是 NaN
assert not pd.isna(result["ma5"]).iloc[4]     # 第 5 个有值
assert pd.isna(result["ma10"]).all()          # MA10 全部是 NaN
```

### 验证结果
✅ 修复后测试通过
